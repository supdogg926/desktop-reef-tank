# DESIGN VISION · 桌面海缸 V2

## 核心理念：Text-as-Icon 艺术方向

桌面海缸 V2 摒弃传统像素精灵图，采用**纯文本即图标（text-as-icon）**的视觉语言。每个海洋生物用彩色 RichTextLabel 渲染，包含中文名 + 性别符号（♂/♀）+ 拉丁学名，通过颜色编码传达物种身份与性别特征。

### 为什么是 Text-as-Icon

- **极低资源消耗**：不需要纹理图集、sprite sheet，减少 GPU 内存占用，适合长时间常驻桌面的 idle 场景
- **视觉独特**：在桌面 overlay 场景中，文本生物比像素图更具识别度，在透明叠加条件下对任何桌面背景都保持可读性
- **信息密度**：文本本身既是"视觉"又是"名字"，不需要额外的 tooltip 或 HUD
- **可扩展性**：新增物种只需添加一行数据定义，无需绘制新素材

### 物种即字体

不同生物使用不同字号传达体型差异：
- 番茄小丑（♂）：24px，小而明亮
- 番茄小丑（♀）：30px，稍大更深
- 彩虹地毯海葵：56px，巨幅视觉中心

## 长期愿景

### 桌面 overlay 生态
- 透明窗口悬浮于桌面之上，像一块真实鱼缸
- borderless + always-on-top，不干扰正常桌面操作
- 占用屏幕底部五分之一，持续可见但低调

### Idle 游戏循环
- 领养 → 观察 → 收集图鉴
- 生物在鱼缸内自主游动（鱼）或轻微摆动（海葵）
- 未来：喂食、成长、繁殖、稀有变异

### 架构路线
- Godot 4.x + GL Compatibility 渲染
- CI 从第一行代码开始（GDScript Lint + GUT 单元测试）
- 数据驱动：CreatureData 词典定义所有物种
- SaveManager / GameManager autoload 管理游戏状态
- 每个 commit 通过完整 CI 管线才合并
